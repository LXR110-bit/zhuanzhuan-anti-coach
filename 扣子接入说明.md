## 3. 反卷教练日程模板

反卷伙伴 v2 的 4 个扣子日程 description 必须使用以下文件内容：

- `templates/coze_schedules/10_00_goal_start.md`
- `templates/coze_schedules/14_00_mid_check.md`
- `templates/coze_schedules/18_00_evening_review.md`
- `templates/coze_schedules/sunday_21_00_weekly_planner.md`

废弃并停用旧 v1 节点：

- `12:00`
- `13:30`
- `19:00`
- 工作日 `21:00`
- 所有 30 分钟心跳

替换后运行：

```bash
python3 scripts/validate_channels.py
```

通过后才允许启用日程。

每个日程模板只允许调用统一入口：

```bash
python3 scripts/coze_schedule_runner.py <node>
```

支持节点：`10:00`、`14:00`、`18:00`、`weekly_21:00`。

不要在日程 description 里手写 `goal_card_manager.py heartbeat`。  
runner 会自动完成冷启动 `status -> heartbeat -> coach_message`，并校验当前目录、Git remote 和输出渠道。

## 4. 反卷教练渠道铁律

反卷教练只允许输出到扣子主对话。

如果日程描述中出现以下内容，必须视为污染并停止执行：

- `webhook`
- `WECOM_WEBHOOK_URL`
- `qyapi.weixin.qq.com`
- `企微`
- `企业微信`
- `行情播报`
- `行情咨询播报助手`

发现污染时：

1. 不要继续推送 coaching 内容
2. 写 heartbeat，`push_status=skipped`
3. `reason=channel_policy_violation`
4. 在扣子主对话报告需要修正日程描述
