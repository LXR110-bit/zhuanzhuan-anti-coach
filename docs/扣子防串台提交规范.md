# 扣子防串台提交规范

目标：防止反卷教练仓再次混入行情代码、行情日志、行情运行数据，避免“日志和代码一起推混”。

## 一、提交前必跑

在反卷教练仓提交前执行：

```bash
python3 scripts/validate_channels.py
python3 scripts/validate_commit_boundary.py
```

`validate_channels.py` 检查日程 description 是否污染企微/webhook。
`validate_commit_boundary.py` 检查 staged files 是否越过仓库边界。

## 二、反卷仓允许什么

反卷教练仓 `zhuanzhuan-anti-coach` 只允许维护：

- skill 主体：`SKILL.md`
- 扣子 6 个日程模板：`templates/coze_schedules/`
- 渠道策略：`policy/channel_policy.json`
- 目标卡脚本和校验脚本：`scripts/`
- 反卷配置：`config/`
- 反卷文档：`README.md`、`docs/`

## 三、反卷仓禁止什么

禁止提交：

- 原始日志：`data/logs/`、`logs/`
- 行情运行数据：`data/daily_report_payload.json`、`data/news_signals*.json`、`data/validation_report.json`
- 行情卡片和趋势：`data/report_cards/`、`data/trend_history/`、`data/daily_price_records/`
- 行情代码或行情仓目录：`zhuanzhuan-AI/`、`行情追踪AI助手/`
- 旧 mono-repo 路径：`skills/anti-involution-coach/`、`skills/anti-pretend-effort/`
- 代码/模板变更和 `data/goal_cards/` 运行态混在同一个 commit

## 四、goal cards 怎么处理

`data/goal_cards/` 是反卷教练运行态。默认不和代码一起提交。

如果确实要留历史，只能单独提交：

```bash
git add data/goal_cards/YYYY-MM-DD.json
python3 scripts/validate_commit_boundary.py
git commit -m "data: archive anti coach goal cards YYYY-MM-DD"
```

## 五、推荐提交拆分

- 代码修复：只提交 `scripts/`、`config/`、`policy/`、`templates/`、`docs/`
- 日程模板：只提交 `templates/coze_schedules/` 和必要文档
- 运行历史：只单独提交 `data/goal_cards/`
- 原始日志：不提交，只在本地 `data/logs/anti_coach_runtime.jsonl` 排查

每次 push 前确认：

```bash
git status --short
python3 scripts/validate_channels.py
python3 scripts/validate_commit_boundary.py
git push origin main
```
