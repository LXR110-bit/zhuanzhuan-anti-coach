# 反卷伙伴 v1 → v2 迁移指南

## 总原则:不要一次性切换

v1 失败的最大教训之一就是"一次上太多变量"。v2 迁移也要分 4 周走。

每周只动一件事,跑顺了再走下一步。**慢就是快**。

---

## 第一周:止血(本周做)

**目标**:停止当前系统对你的消耗,让你能喘口气。

### Step 1:关掉所有 30 分钟心跳(10 分钟)

在 Coze 后台,把以下定时器**全部停用**(不要删,留作回滚):

- 10:30 / 11:00 / 11:30 工作日心跳
- 14:00 / 14:30 / 15:00 / 15:30 / 16:00 / 16:30 / 17:00 / 17:30 工作日心跳
- 19:00 / 19:30 / 20:00 / 20:30 工作日心跳

**保留**:
- 10:00 / 12:00 / 13:30 / 18:00 / 21:00(暂时还是 v1 流程)

### Step 2:本周日晚做一次"周规划"测试(60-90 分钟)

不需要改任何代码,只是手动跟 Coze 主对话做一次。

**怎么做**:
1. 周日 21:00 打开 Coze 主对话
2. 把 `prompts/weekly_planner.md` 整段贴进去,作为 system prompt
3. 跟着 Agent 走 6 阶段(上周回看 → 关键结果 → 任务清单 → 取舍 → 保护时段 → 每日大块)
4. 让 Agent 最终输出一个文本版的"本周作战图"(暂时不需要写 JSON)
5. 把这份作战图存到你能看到的地方(印象笔记 / 飞书文档 / 备忘录都行)

**关键体验**:周一上班时,你应该感觉到"这周该干嘛我心里有数",而不是"今天该干嘛我现想"。

### 本周末做一次小评估

问自己:
- 关掉 30 分钟心跳后,我是更焦虑还是更轻松?(预期:更轻松)
- 周日的周规划耗时多少?有没有触及到"取舍"这一阶段?
- 我是否产出了可执行的"本周作战图"?

如果三个答案都正向 → 进入第二周
如果有一个不正向 → 停留在第一周再跑一周

---

## 第二周:升级日 Agent

**目标**:让 10:00 和 18:00 从 v1 流程切换到 v2 流程。

### Step 1:更新 10:00 prompt(20 分钟)

把 `templates/coze_schedules/10_00_goal_start.md` 内容替换为:

```markdown
# 反卷伙伴 10:00 日校准 v2

输出渠道:coze_only

执行步骤:
1. 进入工作目录
2. 运行 status
3. 读取上周日生成的"本周作战图"(你贴的文本即可,或文件路径)
4. 按 prompts/daily_morning.md 走 3 问:
   - Q1: 今天原计划是 [X],有调整吗?
   - Q2: 今天 3 件必做按优先级?
   - Q3: 深度时段做哪件?
5. 5-10 分钟必须结束

禁止:
- 不要五题澄清(v1 已废弃)
- 不要重新做战略决策
```

### Step 2:更新 18:00 prompt(20 分钟)

把 `templates/coze_schedules/18_00_dinner_handoff.md` 改名为 `18_00_evening_review.md`,内容替换为:

```markdown
# 反卷伙伴 18:00 下班复盘 v2

输出渠道:coze_only

执行步骤:
1. 进入工作目录
2. 运行 status,读今日 goal_card
3. 按 prompts/daily_evening.md 走 5 模块:
   - 今日结果(3 件完成度)
   - 业务观察(1 个,追加到 business_journal)
   - 思维差异(1 个,追加到 thinking_gap_journal)
   - 明日大块(3 件预录)
   - 一句话总结
4. 15 分钟必须结束

禁止:
- 不要鼓励性废话
- 必须捕捉业务观察 + 思维差异(长线复利)
```

### Step 3:删除其他时段定时器

第二周开始,**正式停用**:
- 12:00 工作日(上午收尾)
- 13:30 工作日(下午开工)
- 19:00 工作日(晚上开工)
- 21:00 工作日(下班总结)——晚上 21:00 不再触发,21:00 留给周日的周规划

工作日只剩 **10:00 + 18:00** 两个节点(14:00 中段感知第三周再加)。

### 本周末评估

- 10:00 校准是不是真的 5-10 分钟搞定?如果还是 20-30 分钟,说明 prompt 没限制住,需要再加约束
- 18:00 复盘有没有真的写出业务观察和思维差异?如果只完成了"结果"环节,说明你在跳过最重要的部分,要刻意补
- 本周作战图有没有真的指导日程?如果你每天还是临时想,说明周规划没做透

---

## 第三周:加上 14:00 + 软化人格

**目标**:补全日节奏,把人格从 v1 教练完全切换到 v2 伙伴。

### Step 1:启用 14:00 中段感知(10 分钟)

新建 Coze 定时器 14:00 工作日,prompt 用前面给的模板。

**注意**:这个节点是"可选"的——3 分钟没回应就沉默退出,不要让它变成新的压迫源。

### Step 2:全面更新 SKILL.md

用 `/output/zhuanzhuan-anti-coach-v2/SKILL.md` 替换原 SKILL.md。

**关键差异**:
- 话术风格段全部替换
- 五题澄清(Phase1)+ 三问判定(Phase2)流程描述全部删除
- 加入"清醒伙伴"人格段
- 加入异常处理(累/崩溃/自我攻击)

### Step 3:做一次"主动唤起"测试

在非节点时间(比如某天下午 15:30 突然想复盘),对 Coze 主对话说:
> "复盘一下今天上午"

应该看到的反应:
- ✅ Agent 立刻进入 Phase(读 status,然后按 daily_evening 流程问)
- ❌ 如果还是问"五题"或者"30 分钟干了什么",说明 SKILL.md 没更新到位

### 本周末评估

- Agent 的回应是不是真的"清醒不攻击"?对照对话记录看看
- 你打开 Agent 的心理抗拒是不是降低了?(预期:降低)
- 14:00 节点你回应了几次?如果都没回应,说明这个节点对你不必要,可以删掉

---

## 第四周:完整跑通 + 维护

**目标**:让 v2 系统进入稳态。

### Step 1:维护 my_compass.md

花 30 分钟,基于你的真实情况,填一遍 `config/my_compass.md`:
- 当前阶段(基建期/成长期/加速期)
- 本月关键目标(3 个,具体可衡量)
- 本月主攻的成长重点(从 5 个卡点里选 1-2 个)

**这份文件每月初更新一次,周 Agent 周日会读它**。

### Step 2:数据沉淀的目录建好

```
data/
├── weekly_plans/
│   └── 2026-05-25.json
├── daily_reviews/
│   └── 2026-05-26.md
├── business_journal/
│   └── 2026-05-26.md
├── thinking_gap_journal/
│   └── 2026-05-26.md
└── goal_cards/
    └── 2026-05-26.json
```

### Step 3:第一次完整周循环

- 周日 21:00 周规划 → 输出 weekly_plan.json
- 周一-周五 每天 10:00 / 18:00 跑日 Agent
- 周五 18:00 复盘时多加一句:"本周 3 个关键结果拿到了几个?"
- 下周日 21:00 再做新周规划,**周 Agent 会读到这一周的所有累积**

### 第四周结束的评估

到这里你应该感受到:
- ✅ 每天启动 Agent 不再让你累(因为日 Agent 真的只是"翻账本")
- ✅ 周日规划虽然累但有意义(因为它换来了一周的轻松)
- ✅ 业务感知和思维差异每天都在累积(长线复利)
- ✅ 你打开 Agent 的频率反而提高了(因为不再有抗拒)

如果上面 4 条都达成 → v2 迁移成功
如果有不达成的 → 看具体卡在哪,可能需要再调整(欢迎再来聊)

## 回滚方案

如果 v2 跑不顺,可以分阶段回滚:
1. 留下 v2 的"周规划"和"砍 30 分钟心跳"(这两个是绝对正向的)
2. 日 Agent 暂时回到 v1(五题澄清),但不要每天都做,只在"需要冲刺的关键日"做
3. 18:00 复盘可以简化成手写日记,不一定走 Agent

**核心原则**:**保留 v1 的工程化沉淀(JSON 架构、冷启动协议、防串台),只调 prompt 层**。这样回滚成本极低。

---

## 给你的一句话

v1 失败不是你的失败——是它要求你"每天都做高耗能决策",这件事人本身做不到。

v2 的整个设计哲学是:**把高耗能集中到一个时间点做透,换来日常的轻松和长线的复利**。

如果跑顺了,你会发现:
- 每天打开 Agent 是放松,不是负担
- 每周日的 90 分钟是"被理解 + 被陪伴",不是"被审判"
- 一个月后你回看 business_journal 和 thinking_gap_journal,会看到一份**别人没有的成长资产**

下周日,我等你的第一次周规划。
