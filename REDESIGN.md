# 反卷教练 v2 重构方案

## 一、为什么要改

### 旧版日志暴露的事实(2026-05-21 → 05-23)

| 日期 | 触发节点 | Phase1(create) | Phase2(review) | Phase3(summary) |
|------|---------|----------------|---------------|----------------|
| 5-21 | 18:00 / 19:00 / 21:00 | 0 | 4 | 2(都失败) |
| 5-22 | 10:00 / 12:00 / 13:30 | **0** | 0 | 0 |
| 5-23 | 10:00 / 12:00 / 13:30 | **0** | 0 | 0 |

- **3 天 0 个 goal_card 被创建**——五题澄清一次都没完成
- **8 次系统错误**——verdict 字段类型混淆,说明 Agent 自己也分不清流程
- **触发在跑但人没在跟**——典型的"开始用了→坚持不下去→只剩仪式性触发"

### 用户主观感受

"运转了一周,早上跟它对话觉得有点累了。"
"我感觉我是不是应该再在之前做一件事,就是花一段时间很深刻的把这一周的事都应该想一下,而不是每天现想现挂。"

**这个直觉是对的——根因是单层架构 + 高密度心跳 + 攻击性人格的组合**。

## 二、5 个根本问题

| 问题 | 严重度 | 表现 |
|------|--------|------|
| 1. 缺周锚点,每天从零启动 | ⭐⭐⭐⭐⭐ | 早上 10:00 五题是高耗能决策,每天都做 |
| 2. 心跳密度过高 | ⭐⭐⭐⭐ | 一天 13-14 次触发,30 分钟一次心跳 |
| 3. 教练人格长期消耗 | ⭐⭐⭐ | 24/7 攻击性语气,加剧崩溃倾向 |
| 4. 五题假设单 deliverable | ⭐⭐⭐ | 真实工作是多角色并行,五题不匹配 |
| 5. 假设全天连续工作 | ⭐⭐ | 30 分钟心跳不适配会议/打断密集场景 |

## 三、3 个保留的亮点

- ✅ **JSON 沉淀架构**(goal_card / review / summary 写 JSON,可回看)
- ✅ **冷启动协议**(每次触发先 status 再说话,防失忆)
- ✅ **追问规则 A-E**(虚答识别和反推的细节,有真知)

## 四、双层架构

```
┌─ 月锚 ─────────────────────────────────────────────┐
│  config/my_compass.md (用户手动每月初维护)        │
│  - 本月关键目标 / 在攻什么硬骨头 / 当前成长重点    │
└────────────────────────────────────────────────────┘
                  ↓ 周 Agent 读
┌─ 周 Agent (周日晚 60-90 分钟,1 次) ───────────────┐
│  深度对话,产出 weekly_plan.json:                  │
│  - 本周 3 个关键结果                                │
│  - 角色时间预算 (执行/策划/优化/学习/协作)         │
│  - 每天大块时间分配                                 │
│  - 本周"主动取舍"(砍/简化/委托)                  │
│  - 本周"重要不紧急"保护时段                      │
└────────────────────────────────────────────────────┘
                  ↓ 日 Agent 读
┌─ 日 Agent (每天 2-3 个节点,轻量) ─────────────────┐
│  10:00 开机校准 (5-10 分钟)                        │
│    └─ 翻账本式 3 问,基于 weekly_plan              │
│  14:00 中段感知 (3 分钟,可跳过)                   │
│    └─ 上午 3 件必做完成度 + 业务异常记录          │
│  18:00 下班复盘 (15 分钟,核心)                    │
│    └─ 结果 + 思维差异 + 业务观察 + 明日大块       │
└────────────────────────────────────────────────────┘
```

### 触发次数对比

| 维度 | 旧版 | 新版 |
|------|------|------|
| 每周高耗能决策 | 7 次(每天五题) | 1 次(周日深度) |
| 每天触发节点 | 13-14 次 | 2-3 次 |
| 心跳间隔 | 30 分钟 | 4 小时 |
| 周累计耗时 | 难统计,但很累 | 周 90 分钟 + 每天 25 分钟 = 约 4 小时 |

## 五、人格调整:从"反卷教练"到"清醒伙伴"

| 维度 | 旧版 | 新版 |
|------|------|------|
| 核心姿态 | 教练训斥下属 | 合伙人提醒你 |
| 语气 | 攻击、怼、审判 | 直接、清醒、不奉承 |
| 反馈 | 几乎只有负反馈 | 识别进展 + 指出问题 |
| 对虚答 | "再答一遍!" | "我需要具体的,换个问法:..." |
| 对崩溃 | (无机制) | 允许"今天压力大,简化版"模式 |

**保留的硬核**:逼具体、不接受"在思考"、识别陷阱模式、防过度准备
**去掉的攻击性**:训斥语气、否定句、"失败了""又陷进去了"这类话

## 六、新增"业务感知"维度

每天 18:00 复盘强制 1 个字段:`business_observation`

- 1 个数据观察 / 用户反馈 / 异常 / 疑问
- 沉淀到 `data/business_journal/YYYY-MM-DD.md`
- 周 Agent 周末会读一周的 journal,帮你看出趋势

**这把"业务感知系统"无缝整合进 Agent 框架,不需要额外搭系统**。

## 七、迁移路径(不要一次性切换)

### 第一周(本周):**只换两件事**
1. **关掉所有 30 分钟心跳**(只保留 10:00 / 14:00 / 18:00 / 21:00)
2. **周日晚做一次"周规划"对话**,用新的 weekly_plan prompt

**目标**:体验"周决策做透 + 日决策翻账本"的差异。**不要急着改其他东西**。

### 第二周:**升级日 Agent prompt**
1. 把 10:00 五题换成"3 问翻账本"
2. 把 18:00 改成 15 分钟复盘(含业务观察字段)

### 第三周:**软化人格**
更新 SKILL.md 的话术风格,从"教练"过渡到"清醒伙伴"

### 第四周:**完整跑通**
- 月初维护 my_compass.md
- 周日做周规划
- 工作日跑日 Agent
- 周五做周复盘
